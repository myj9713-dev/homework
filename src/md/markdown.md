# 마크다운 문법

# 마크다운(Markdown)이란?

텍스트 기반의 마크업언어로 2004년 존 그루버에 의해 만들어졌으며, 특수기호와 문자를 이용한 매우 간단한 구조의 문법을 사용하여 웹에서도 보다 빠르게 컨텐츠를 작성하고 직관적으로 인식 할 수 있으며, HTML로 변환이 가능하다.

> 주로 GitHub의 README 파일, 블로그 포스팅, 메모 앱 등에서 사용

1. 장점

- 별도의 도구가 필요 없다.
- 텍스트(Text)로 저장되서 용량이 적어 보관이 용이하다.
- 텍스트파일이기에 버전관리시스템으로 변경이력을 관리할 수 있다.
- 지원하는 프로그램과 플랫폼이 다양하다.

2. 단점

- 표준이 없어서 도구에 따라서 변환방식이나 생성결과물이 다르다.
- 모든 HTML 마크업을 대신하지 못한다.

# 마크다운 사용법(문법)

## 헤더(Hearders)

제목은 크기가 다르게 표시되며, 샵기호(#) 뒤에 제목을 입력하는 방식으로 총 6개까지만 작성할 수 있다.

```
# 제목 1 (가장 큰 제목)

## 제목 2

### 제목 3

#### 제목 4

##### 제목 5

###### 제목 6 (가장 작은 제목)
```

# 제목 1 (가장 큰 제목)

## 제목 2

### 제목 3

#### 제목 4

##### 제목 5

###### 제목 6 (가장 작은 제목)

## 목록

### 순서있는 목록

숫자와 점을 사용하는데, 목록에서 첫 번째 항목에 입력한 번호 기준으로, 첫 순서가 1이면 (1,2,3...), 첫 순서가 5면 (5,6,7...) 오름차순으로 표시한다.

```
1. 첫 번째 항목
2. 두 번째 항목
3. 세 번째 항목
```

1. 첫 번째 항목
2. 두 번째 항목
3. 세 번째 항목

```
5. 다섯 번째 항목
3. 여섯 번째 항목
1. 일곱 번째 항목
```

5. 다섯 번째 항목
6. 여섯 번째 항목
7. 일곱 번째 항목

### 순서없는 목록

블릿(bullet)기호의 세 가지, (\*, +, \_) 뒤에 공백 하나를 넣어서 사용한다.

```
* 별표(Asterisk)
+ 더하기(Plus Sign)
- 하이픈(Hyphen/Dash)
```

- 별표(Asterisk)

* 더하기(Plus Sign)

- 하이픈(Hyphen/Dash)

### 목록 중첩(하위 목록 만들기)

하위 목록을 만들 때는 부모 목록 기호(또는 숫자)에서 공백 2칸 또는 4칸을 넣어서 사용한다.

```
- 부모목록
    - 하위목록1
    - 하위목록2

1. 부모목록
    1. 하위목록1
    2. 하위목록2
```

- 부모목록
  - 하위목록1
  - 하위목록2

1. 부모목록
   1. 하위목록1
   2. 하위목록2

### 체크리스트(Task List)

할 일 목록이나 작업 상태를 표시할 때 순서 없는 목록 기호 뒤에 대괄호([])를 사용한다.

```
- [x] 할 일 1(완료)
- [ ] 할 일 2(미완료)
```

- [x] 할 일 1(완료)
- [ ] 할 일 2(미완료)

## 폰트 서식 및 강조

텍스트의 모양을 바꾸어 강조를 하고 싶을 때 사용한다.

```
**굵게** 또는  __굵게__
*기울임* 또는 _기울임_
***굵고 기울임***
~~취소선~~
```

**굵게** 또는 **굵게**
_기울임_ 또는 _기울임_
**_굵고 기울임_**
~~취소선~~

## 인용문(Blockquote)

텍스트를 강조하고 싶거나 본문과 시각적으로 구분하고 싶을 때 (>)기호를 사용한다.

- 기본 인용문

```
> 이것은 인용문입니다.
```

> 이것은 인용문입니다.

- 중첩 인용문

```
> 첫 번째 인용문입니다.
>
>> 두 번째 중첩 인용문입니다.
>>
>>> 세 번째 중첩 인용문입니다.
```

> 중첩된 인용문도 가능합니다.
>
> > 두 번째 중첩 인용문입니다.
> >
> > > 세 번째 중첩 인용문입니다.

- 인용문 내 다른 서식 사용(여러줄 인용 가능, 각 줄 앞에 (>)기호 추가)

```
> ## 인용문 제목
> 이것은 **강조된 내용**이 포함된 인용문입니다.
> *이것은 기울인 내용의 인용문입니다.*
```

> ## 인용문 제목
>
> 이것은 **강조된 내용**이 포함된 인용문입니다.
> _이것은 기울인 내용의 인용문입니다._

## 코드

백틱(`)기호를 사용해 코드나 명령어를 표시한다.

### 코드 블록

여러 줄의 긴 코드를 본문과 완전히 분리하여 표시할 때 사용한다. 이 방법으로 코드를 복사하기 쉽게 별도의 상자가 만들어진다.

```
여기는 코드 블록 방식으로 분리된 공간입니다.
여기서 코드를 복사하여 쉽게 사용 할 수 있습니다.
```

## 링크 및 이미지

### 링크

링크는 문법 \[이름](링크"선택적 툴팁") 순서로 추가 할 수 있다. 참조링크를 그대로 사용하고 싶다면, 일반적인 URL이나 꺽쇠 괄호(< >)안에 URL은 자동연결된다.

> 선택적 툴팁은 마우스를 올렸을 때 나타나는 툴팁(Tooltip)이다.

```
[네이버](www.naver.com"네이버 홈페이지로 이동")
구글링크: <www.google.com>
```

[네이버](www.naver.com"네이버 홈페이지로 이동")
구글링크: <www.google.com>

### 이미지

문서에 시각적인 요소를 삽입할 때 사용한다. 문법은 기본 링크 문법 앞에 느낌표(!)를 붙인다.

```
![고양이](https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcQzmi_ortuZYOlzXsZpjUWEDP2i04S3v1drJw&s"고양이사진입니다")
```

## 기타 요소

### 수평선

섹션을 시각적으로 분리시킨다.최소 3개 이상 연속으로 사용해야하며, 기호들 사이에 공백이 들어가도 된다. 또한 수평선 문법의 위아래로 빈 줄로 분리하는 것이 안전하다.

```
 첫 번째 달락입니다.

 * * *

 두 번째 달락입니다.

 ***
 *****
 - - -
 ----------
```

첫 번째 달락입니다.

---

두 번째 달락입니다.

---

---

---

---

### 표

데이터를 구조화하여 표시할 때 유용하다. 파이프 기호(|), 하이픈(-)을 사용하여 행과 열을 구분한다.

```
| 제목1  | 제목2  |  제목3 |
| :----- | :----: | -----: |
| 좌정렬 | 가운데 | 우정렬 |
| 내용1  | 내용2  |  내용3 |
```

| 제목1  | 제목2  |  제목3 |
| :----- | :----: | -----: |
| 좌정렬 | 가운데 | 우정렬 |
| 내용1  | 내용2  |  내용3 |

### 줄바꿈

한 문단 내에서 시각적으로만 줄을 바꿔야 할 때 사용된다. 방법은 문장의 끝에 공백 두 번 입력후 엔터를 누르면 된다.

```
첫 번째 줄에서 공백 두 개
두 번째 줄로 바뀝니다.
```

첫 번째 줄에서 공백 두 개  
두 번째 줄로 바뀝니다.

### 백플래시

마크다운 문법에서 특수 기호들을 일반 텍스트로 표시하고 싶을 때, 그 기호 바로 앞에 백플래시를 넣어 입력한다.

```
\- 백플래시를 넣으면 목록이 안됩니다.
- 넣지 않으면 목록이 됩니다.
```

\- 백플래시를 넣으면 목록이 안됩니다.

- 넣지 않으면 목록이 됩니다.
